# MANIFEST.txt — paths included in the encrypted credential vault.
#
# Two path classes:
#   ${WORKSPACE}/...   workspace-relative. Resolved against $TRUESIGHT_WORKSPACE
#                      (default: $HOME/Applications). Each governor sets this
#                      env var if their repos live somewhere other than the
#                      default. Restore re-roots these to the restoring user's
#                      workspace, so backing up at ~/Applications/dao_client/
#                      and restoring with TRUESIGHT_WORKSPACE=~/code lands the
#                      file at ~/code/dao_client/.env. (V1.1)
#   ~/...              home-relative. Resolved against $HOME on both backup and
#                      restore. Use for dotfiles, system creds, etc.
#
# Lines starting with `#` and blank lines are ignored. Paths may be files OR
# directories. Missing paths are warned, not fatal — manifest is a forward
# declaration.
#
# To add/remove: edit this file, then run scripts/backup.sh manually to
# produce a fresh snapshot. The next launchd-triggered run picks up the
# change automatically; for new WORKSPACE entries also re-run install_agent.sh
# so the WatchPaths in the launchd plist are regenerated.
#
# See README.md for the threat model + restore runbook; ONBOARDING.md for the
# LLM-runnable new-governor flow.
